# IMS UI Pattern Inventory

This file maps reusable UI patterns from `ims/IMS Dashboard.html` into the production app.

## Shared shell patterns

- Sidebar with module-first navigation, active highlight, and compact labels.
- Top bar with search, notification bell, profile awareness, and quick context title.
- Token-driven visual language: dark surfaces, accent gradients, soft borders, rounded cards.

## Reusable interaction patterns

- Select-and-detail pattern (table/list on left, contextual pane on right).
- Feed/timeline pattern for notifications and audits.
- Status-first badges for priority, workflow stage, and approvals.
- Inline action clusters for approve/reject/mark-ready operations.

## Module mapping

- Dashboard: live KPI + active jobs, quick-open details.
- Clients: list + detail pane + create/edit form.
- Jobs: board/grouped list, task add/update, review-link generation.
- HR: employee cards, attendance action, leave queue with decisions.
- Creative: task picker, version uploads, review readiness.
- Accounts: KPI strip, invoice list, payment/reminder actions.
- Notifications: unread filters + mark read/mark all read.
- Leadership: KPI and category breakdown.
- Approvals: mixed pending queue by domain (task/leave/finance).
- Audit: chronological event feed with filtering.
